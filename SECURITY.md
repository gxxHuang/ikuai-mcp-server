# 安全策略

| 版本 | 支持状态 |
|------|---------|

## 报告漏洞

如果你发现安全漏洞，**请不要在公开 Issue 中报告**。

请发送邮件到 **qqhuangweimin@gmail.com**，我们会尽快回复。

### 期望内容

- 漏洞类型和位置
- 复现步骤
- 潜在影响范围
- 建议修复方案（可选）

### 处理流程

1. 确认收到报告（48 小时内）
2. 评估严重性
3. 开发修复
4. 发布补丁 + CVE（如适用）
5. 公开披露（修复后 30 天）

## 安全检查清单

- [ ] `.env` 文件不会提交到 Git
- [ ] 危险操作需要 `confirm=True` 二次确认
- [ ] 凭据不在日志中输出
- [ ] 路由器密码仅通过环境变量注入
